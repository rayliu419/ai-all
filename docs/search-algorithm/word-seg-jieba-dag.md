## 分词传统方法：jieba 的 DAG 分词

### jieba 解决的核心问题

> 给你一本词典，怎么把句子切成词。

词典里有的词 → 用 DAG + DP 切出来
词典里没有的词（OOV）→ 交给 HMM 补上

所以**想在一个领域分得准，关键就是提供一本好的领域词典**。

---

### 词典长什么样

jieba 的词典是一堆 `词 + 频率`：

```
我  123
在  234
学习  456
结巴  345
分词  456
结巴分词  23
```

频率来自已分词语料的统计结果（比如 1998 年《人民日报》语料库），**频率越高，分词时越容易被选中**。

---

### 词典怎么用

分三步走：

**1. 构建前缀词典** — 把每个词的所有前缀也注册进去

"结巴分词"会注册"结"、"结巴"、"结巴分"、"结巴分词"。"结巴分"在原始词典里没有，但作为前缀自动加入（freq=0）。**freq=0 的前缀充当"桥"，让扫描可以穿过非词区域找到后面的长词**。

**2. 构建 DAG — 找出所有可能的切分方式**

从每个字出发，每次多吃一个字，查前缀词典。freq>0 就记下来，freq=0 就跳过但继续往后扫。

以"我在学习结巴分词"的位置 4（"结"）为例：

```
前缀词典中有：
结       freq=234        是真实词
结巴     freq=345        是真实词
结巴分   freq=0          是前缀（自动生成），不是真实词
结巴分词 freq=23         是真实词

扫描过程：
步1: "结"       → ✅ freq>0  → 记终点 4
步2: "结巴"     → ✅ freq>0  → 记终点 5
步3: "结巴分"   → ✅ freq=0  → 不记，继续往后扫
步4: "结巴分词" → ✅ freq>0  → 记终点 7
步5: "结巴分词X"→ ❌ 不在词典 → 停止

→ DAG[4] = [4, 5, 7]
```

每个位置都这样扫一遍，就得到完整的 DAG：

```
0:我 → [0]
1:在 → [1]
2:学 → [2, 3]
3:习 → [3]
4:结 → [4, 5, 7]
5:巴 → [5]
6:分 → [6, 7]
7:词 → [7]
```

从 0 到 7 的所有路径，就是所有候选分词方案。

**3. DP 选最优路径** — 从右往左，按词频概率选得分最高的

```
位置7"词"   → 只有"词"，记下得分
位置6"分"   → 可选"分"或"分词"，选概率大的
位置4"结"   → 可选"结"、"结巴"、"结巴分词"，选概率最大的
...
```

最终选出：**我 → 在 → 学习 → 结巴分词**

如果词典里没有"结巴分词"（或频率很低），结果就是：

**我 → 在 → 学习 → 结巴 → 分词**

这就是领域词典的作用——**你调整词频，它调整切分结果**。

---

### 词典覆盖不到的怎么办

单字序列（DAG 和 DP 处理不了的组合）交给 HMM + Viterbi 做序列标注，识别未登录词。这是辅助手段，领域词典覆盖率越高，需要 HMM 兜底的情况就越少。

---

### 一句话

> **jieba 分词 = 给你一本词典帮你把句子切好，词给得准就切得准。**
