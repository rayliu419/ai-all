第一幕

🧠 神经网络的函数选择 — 每个变换都在编码一个决定

神经网络是一系列函数（变换）的组合。但选择什么函数从来不是任意的——每个激活函数、每个规范化操作，都在编码一个设计者对于"这个世界是什么样的"的假设。

选择函数，就是在选择满足特定性质的变换。这些性质可以归为两个维度。

1

数学运算的要求

—— 约束来自"可行"

  • 输出必须能解释为概率（softmax）
  • 梯度不能爆炸 / 消失（求导链式法则的约束）
  • 数值稳定性、尺度不变性、零中心化

这些是必要条件。不满足，网络就训练不好。

2

数学上的意义

—— 追问"为什么这个函数能代表现实规律？"

  • 拓扑学意义：ReLU 折纸般折叠高维空间
  • 统计学意义：sigmoid 模拟神经元激活概率
  • 先验知识：CNN 注入"局部相关性"假设

这些是深刻的设计哲学。理解了它，你就能理解为什么有些架构天然更好。

🏔️ 深潜一：ReLU 为什么更好？—— 一段函数筛选史

今天我们知道 ReLU 比 sigmoid 好，但当初并不是这样推理出来的：

  1. 早期（数学驱动）：sigmoid 的灵感来自生物神经元的"激活阈值"——看起来有生物学依据，数学上也光滑可导，所以被广泛使用
  2. 中期（工程试错）：人们发现深层网络用 sigmoid 训练不动了——梯度在链式法则下一层层地消失。ReLU 的常数导数（x > 0 时导数为 1）完美解决了这个问题，但当时并不完全清楚为什么它"这么好"
  3. 后期（理论解释）：数学和优化理论给出了优雅的解释——ReLU 对高维空间的"折纸"操作为什么从根本上改变了数据分布的拓扑结构
启示：选择激活函数，绝大部分时候就是在选择它的导数。基于梯度下降的神经网络中，导数特性决定了训练成败。

🔬 ReLU 的拓扑学意义

ReLU 在数学上相当于对高维空间进行弯折（folding / creasing）。

每一个 ReLU 神经元都像是一把"剪刀"，把空间沿着某条直线折叠。成千上万个 ReLU 组合起来，就能把原本线性不可分的高维空间，像折纸一样层层折叠、裁剪，最终让不同类别的数据在新的空间里变得线性可分。

这不仅仅是一个工程技巧——这是在用几何变换重新组织信息的表达方式。

⚖️ 深潜二：偏置（Bias）—— 一个"小参数"背后的几何革命

💡
核心直觉：如果只有 $W\vec{x}$，原点 $(0,0,...,0)$ 永远被固定死。无论怎么旋转、拉伸，原点变换后依然是原点（$W \cdot \vec{0} = \vec{0}$）。但在现实任务中，分类边界几乎从不经过原点。偏置 $\vec{b}$ 的唯一作用就是平移整个高维空间，赋予网络脱离原点束缚的能力。

🤔 偏置能用矩阵替代吗？

可以——通过齐次坐标（Homogeneous Coordinates）给向量升一维，把平移融入矩阵乘法。

但深度学习框架仍然显式写 +b：

  • 显式表达：权重（敏感度）和偏置（基准线）分开，更符合直觉
  • 节省内存：强行拼接 1 到数据中徒增存储，广播机制在 GPU 上更高效

🔗 偏置的神经网络解释

在 sigmoid 时代，偏置直接决定了神经元是否容易被激活——当输入加权和超过偏置阈值时，神经元才"兴奋"。

偏置越大，神经元越难激活（阈值更高）。这也是早期神经科学启发的一个设计痕迹。

· · ·

🏗️ 不把网络当黑盒

理解这些"为什么"的实用价值：当模型出问题时，它指导你从哪个方向优化。

反例：梯度消失 → 你知道该检查激活函数和初始化方式，而不是盲目调学习率。

正例：CNN 包含的先验知识（局部性 + 平移不变性）让它能以数量级更少的数据达到相同的效果。

📌 第一幕总结

神经网络的函数选择不是随意的——每个变换都在编码一个关于世界的假设。数学可行性和哲学理解是两条并行的线索，交汇在一起决定了"什么架构能 work"。理解这些，是走出"调参工程师"的第一步。

第二幕

📐 梯度下降 — 是怎么学的，和为什么学不会是两回事

🎬 3Blue1Brown — 梯度下降（视频）

梯度下降是神经网络学习的机制，但它不保证学到有意义的东西。接下来的实验会暴露这个深刻的区别。

⚙️ 机制：梯度下降简述

  • 单变量函数 → 求导找最小值
  • 多变量函数 → 梯度是一个向量，指向"最陡上升"方向，取反就是下山方向
  • loss function 定义"错误程度" → 梯度告诉你如何调整参数来减小错误
⚠️ 这是多变量微积分的内容，如果三维梯度的几何意义不清晰，值得停下来补充一下。这是所有后续理解的基础。
↓

❌ 实验：两层的数字识别网络 — 它没有按预期工作

😤 期望 vs 现实
🟢 我们期望的

第一层识别"横、竖、撇、捺"等笔画部件，第二层组合这些部件来识别数字。层级结构**有意义的**抽象。

🔴 实际发生的

用随机图片测试，网络给出错误输出。特征可视化显示第一层看到的不是笔画，而是噪音般的模糊鬼影——它根本没学会我们想象中的"特征"。

梯度下降在运行，loss 在下降，但网络学到的不是我们想要的东西。这就是机制正确但意义缺失。

↓ 为什么？三个根本原因 ⟶

🔍 诊断：三个原因解释为什么"学好"不等于"学会"

1

全连接的"全局性"摧毁了空间局部性

——每个神经元看整张图

  • 现实：每个隐藏神经元与所有像素相连
  • 后果：神经元不关注"局部"，而是全局统计。最终学到的不是笔画边缘，而是斑驳光点

→ 教训：没有局部性假设，模型只能找粗糙的统计规律

2

浅层网络（仅两层）缺乏"抽象深度"

——像素→边缘→纹理→部件→整体的层次需要多层

  • 两层网络没有足够层级来实现分层抽象
  • 它"走捷径"：直接拟合像素→标签的非线性映射，而不是构建有意义的中间表示

→ 教训：深度不是为了"更深的网络听起来更厉害"，而是层次结构是抽象的必要条件

3

缺乏"平移不变性"

——"7"向右移 2px，对网络来说就是全新输入

  • 全连接网络对位置极其敏感：换个位置，之前学到的权重完全对不上
  • 没有不变的图像，只有无穷多的变体

→ 教训：没有不变性假设，需要指数级数据来覆盖所有变体

💊 解法：CNN 如何用"结构先验"同时解决这三个问题

CNN 之所以大幅超越全连接网络，不是因为它"更聪明"，而是因为它在设计之初就注入了关于图像世界的物理假设：

🏋️ 权重共享

同一个卷积核在整个图像上滑动，参数只有一小套。这不仅减少了参数量，更关键的是——无论特征出现在画面的哪个位置，都能被同一组参数检测到。

🧠 归纳偏置（Inductive Bias）

模型在训练前就被注入了两大假设：

  • 局部相关性——靠得近的像素关系更密切
  • 平移不变性——同一物体换个位置还是同一物体
CNN 在训练开始之前，已经站在了终点线附近。
🔗
这里完美呼应了 how2ai-course 的内容：找到数据的不变性能大幅减少所需样本量。CNN 找到的不变性是"平移"——所以它不需要像 MLP 那样，用成千上万张"往右平移 5 像素"的图片去肉搏学习这个规律。这不仅仅是效率提升，这是学习范式的根本转变。
📌 第二幕总结

梯度下降是学习的引擎，但架构才是学习的方向盘。

两层的全连接网络在数字识别上的失败揭示了一条核心规律：没有结构先验的模型，需要用指数级的数据和参数来弥补架构的盲目性。CNN 的成功不是因为它"更深"或"更新"，而是因为它编码了对这个物理世界的基本假设——而这些假设，正是我们通过"理解数据、找到不变性"得到的洞察。

第三幕

🔙 反向传播（Back Propagation）— 学习的"幕后推手"

🎬 3B1B — 反向传播（直觉篇） 🎬 3B1B — 反向传播（微积分篇） 📺 B站 — 反向传播微积分

所谓的学习，就是找到所有参数组合，使得 loss function 最小。

对于手写识别神经网络，我们需要求 13000 个参数的负梯度。虽然参数分布在不同的网络层次上，但我们可以先把它们简单地看作一个梯度——每个参数对 loss function 的敏感程度不一样。

举个例子：如果一个权重是 3.2，另一个是 0.1，则第一个有扰动时对 loss 的影响大约是第二个的 32 倍。敏感度差异巨大。

↓ 让我们通过"数字 2"的故事来理解反向传播的直觉 ⟶

👁️ 直觉：识别数字"2"背后的三个动作

在输出层，网络要对"这张图是不是 2"做出判断。反向传播会依次做三件事，每一件都有清晰的直觉逻辑：

1

改变 Bias

直接给"2"的输出神经元加温

Bias 就像一个神经元的"基础门槛"或"自带分值"。如果不管输入是什么，我都想让"2"的概率变大，最直接的方法就是把"2"这个神经元的 Bias 调大——相当于降低激活"2"的门槛。

2

改变 Weight

顺藤摸瓜，给贡献大的前层神经元"发奖金"

"2"的输出是由倒数第二层的神经元乘以各自的权重相加得到的。如果倒数第二层某个神经元 $A$ 正处于高激活状态（比如它捕捉到了"2"底部的横线），把 $A$ 连接到"2"的权重调大，就能极大地放大这个正向刺激。

"大的权重调整起来更有性价比"——因为权重调整幅度正比于前层激活值。前层越兴奋，调整它的权重效果越立竿见影。
3

改变前一层激活值

对前一层提要求——"有功的再兴奋一点，帮倒忙的闭嘴"

光改当前参数（Bias 和 Weight）是有上限的。反向传播最厉害的地方在于，它会向倒数第二层传达一个愿望（Desire）：

  • 对连接着正权重的神经元 → "再亮一点"（激活值变大）
  • 对连接着负权重的神经元 → "再暗一点"（激活值变小）
💡
贯穿逻辑：这三步本质上是在回答一个问题——"如果要降低 loss，每个参数应该往哪个方向调整多少？" Bias 直接调门槛，Weight 放大关键信号，激活值调整则为下一轮的反向传播做好了准备。每一层都做同样的事情，信号就从输出层一层层传回了输入层。

⚡ mini-batch 与 SGD

实际训练中，不会只用一个样本的梯度来更新参数——那样太毛糙（每个样本的"愿望"都不同）。

相反，用一小批（mini-batch）样本的平均梯度来决定更新方向。这就是随机梯度下降（SGD）——在效率和稳定性之间取平衡。

🔬 反向传播的微积分 — 从扰动到链式法则

直觉给了我们方向，但微积分给了我们精确的数字。

反向传播本质上就是微分的链式法则在神经网络中的应用。以下用"多条数轴联动"和"微小扰动（Nudges）"的直观方式来解释。

📐 核心直觉：导数 = 对局部扰动的敏感度

三条基本认识

  • $\frac{dy}{dh}$：如果系统在 $h$ 这个位置被轻轻拨动了 $dh$，最终的输出 $y$ 会产生多大的波动？—— 这是 $y$ 对 $h$ 的直接敏感度。
  • $\frac{dy}{dx}$：如果源头 $x$ 被拨动了 $dx$，经过中间所有多米诺骨牌的连锁反应后，最终的 $y$ 会产生多大的波动？—— 这是 $y$ 对 $x$ 的全路径敏感度。
  • 为什么需要"不同的中间增量"来表达？ 因为在实际面对复杂系统时，我们往往无法一眼看穿最底层（$x$）和最顶层（$y$）的关系，但我们能看清相邻变动的关系。链式法则就是把这些"相邻片段"串起来。
🧩
在神经网络中的映射：$x$ 是输入的图像 → 中间隐藏神经元是无数 $h_1, h_2, ...$ → $y$ 是最终的预测误差 Loss。为了调整网络，我们需要知道"每一个中间的神经元如果产生扰动，对最终的 Loss 会有什么影响"——这正是链式法则解决的问题。

🧮 极简网络举例

从"每层只有一个神经元"的极简网络开始：

$a^{(L-1)} \xrightarrow{\times w^{(L)} + b^{(L)}} z^{(L)} \xrightarrow{\sigma(z)} a^{(L)} \xrightarrow{\text{计算与 } y \text{ 的差距}} C_0$
  • $a^{(L-1)}$：前一层神经元的激活值
  • $z^{(L)} = w^{(L)} a^{(L-1)} + b^{(L)}$：当前的加权输入（Weighted sum）
  • $a^{(L)} = \sigma(z^{(L)})$：当前层的激活值（$\sigma$ 为激活函数，如 Sigmoid 或 ReLU）
  • $C_0 = (a^{(L)} - y)^2$：单个样本的代价（$y$ 是标签，即期望输出值）

扰动传导链

如果我们对 $w^{(L)}$ 微调一个极其微小的量 $\partial w^{(L)}$，这个扰动会像水波一样向右传导：

$\partial w^{(L)} \rightarrow \partial z^{(L)} \rightarrow \partial a^{(L)} \rightarrow \partial C_0$

根据链式法则，代价 $C_0$ 对权重 $w^{(L)}$ 的偏导数（敏感度）就是这三级跳的乘积：

$\displaystyle \frac{\partial C_0}{\partial w^{(L)}} = \frac{\partial C_0}{\partial a^{(L)}} \cdot \frac{\partial a^{(L)}}{\partial z^{(L)}} \cdot \frac{\partial z^{(L)}}{\partial w^{(L)}}$

推导最后一层到前一层，和前一层到更前一层是类似的——递归地应用同样的逻辑。

复杂网络的推导跟极简网络也是类似的——只是多写了下标。
📌 第三幕总结

反向传播是神经网络学习的引擎：它用微分的链式法则，把输出层的误差信号逐层反向传播到每一个参数，告诉我们"每个权重该往哪个方向调整多少"。直觉层面的三个动作（调 Bias、调 Weight、调前层激活值）和微积分层面的链式法则是同一件事的两种语言——一个让你理解"为什么"，一个让你算出"是多少"。

🔧 附章一

📊 数学工具箱 — 矩阵乘法的两种灵魂

在深入 Transformer 之前，有一个贯穿始终的数学工具需要先建立直觉：矩阵乘法有两种理解方式，它们服务于不同的目的。

理解这两种视角，是看懂 Attention 和 MLP 数学内在的基础。

🧩 视角一：列向量 —— 线性组合

核心思想

矩阵乘以向量，是矩阵各列向量的线性组合。

$A = \begin{bmatrix} | & | \\ v_1 & v_2 \\ | & | \end{bmatrix}$, $x = \begin{bmatrix} x_1 \\ x_2 \end{bmatrix}$ → $Ax = x_1 v_1 + x_2 v_2$

结果向量由矩阵的列向量加权求和得到。权重就是输入向量 $x$ 的对应分量。

为什么需要这个视角？

  • 空间生成与基底：结果向量 $b$ 必须处于 $A$ 的列空间（Column Space）中，方程 $Ax=b$ 才有解。这告诉我们结果落在什么空间里。
  • 坐标变换：把 $A$ 的列向量看作一组"新坐标轴"（基底），向量 $x$ 就是在这组新坐标轴下的坐标。$Ax$ 把新坐标系下的向量转换回标准坐标系。

🎯 视角二：行向量 —— 点积相似度

核心思想

矩阵乘以向量，是矩阵各行向量与该向量的点积（内积）。

$A = \begin{bmatrix} — & r_1 & — \\ — & r_2 & — \end{bmatrix}$, $x$ → $Ax = \begin{bmatrix} r_1 \cdot x \\ r_2 \cdot x \end{bmatrix}$

每个输出的分量分别来自输入向量与矩阵对应行向量的点积——也就是相似度分数。

为什么需要这个视角？

在深度学习中，计算点积就是计算语义相似度：

  • Attention 的 $QK^T$ 就是用行向量视角——每个 Query 行向量与所有 Key 列向量计算相似度
  • MLP 升维矩阵 $W_1$ 也是用行向量视角——每个行向量编码一个"事实的 Key"，与输入做点积来检查匹配度
🔗
两视角的统一：同一个矩阵乘法，从行看是"匹配模式"，从列看是"组合基底"。Attention 用前者找谁相关，MLP 用前者检索知识、用后者组装结果。理解了这一点，后面的内容会豁然开朗。
第四幕

🔄 注意力机制 — 让每个词看见整个句子

🏗️ 前置：把 Transformer 看成一个矩阵工厂

在最高抽象层，Transformer 做的事情很简单：把模型看成一组可调参数的矩阵，这些矩阵作用于从待处理数据提取的向量。

所有矩阵都是在预训练阶段通过海量文本学习得到的——它们学会了如何把"词"变成"有上下文的表示"。

📐 重要函数：Softmax

Softmax 的作用是把任意一组实数转换为概率分布：

  • $e^x$：把正负数都映射为正数
  • 分子分母归一化：所有输出之和为 1
$\text{Probability}_i = \frac{e^{z_i / T}}{\sum_j e^{z_j / T}}$

🌡️ Temperature 参数

LLM 在预测下一个词时，会为词表中所有词计算一个概率分布。Temperature $T$ 控制分布的"尖峭"程度：

  • $T$ 很小：原本稍微领先的分数被指数级放大，形成"赢家通吃"——输出确定但呆板
  • $T$ 很大：所有词的概率被拉平——输出多样但可能不相关

Softmax 之前的输出称为 Logits（未归一化的分数）。

📦 嵌入矩阵（Embedding Matrix）$W_E$

语句中的每个词被转换为一个高维向量：

  • 单词被映射为高维空间中的点
  • 空间中的方向编码了语义——"男人" - "女人" 可能就编码了性别信息
  • 向量的最近邻居往往是语义近似的词
  • 向量的点积反映了语义相关度

在后续运算中，每个 token 的 embedding 还会融入丰富的上下文信息。

🔄 去嵌入矩阵（Unembedding Matrix）$W_U$

与 $W_E$ 作用相反——把经过多层注意力 + MLP 处理的最终向量映射回词汇表，得到每个词作为下一个词的分数（Logits）。

🎯 Attention 的核心直觉

假设某个 token 有一个原始的高维向量 $E_i$（在进入特定训练批次前，它是固定的初始向量）。它仅包含它自己的信息（如果加了位置编码，还包括位置信息）。

注意力机制的目标：通过加权吸收其他 token 的信息，把 $E_i$ 调整为包含上下文的新向量 $E_i'$。

$E_i \rightarrow E_i'$

🔑 QKV：Attention 的三个角色

Q

Query

"我在找什么？"

$E_i \rightarrow Q_i$。把当前词向量转成一个"查字典的问题"——一个相对低维的向量。

K

Key

"我有什么？"

$E_i \rightarrow K_i$。把每个词向量转成一个"被查的索引"——和 Q 同维度的向量（因为要做点积）。

V

Value

"找到了怎么改？"

$E_i \rightarrow V_i$。如果被判定"相关"，就贡献自己的内容方向来构造 $\Delta E$。

💡
简单的类比：你在图书馆查书。Query = 你心里的问题，Key = 书的目录索引，Value = 书的内容。你通过问题和目录的匹配找到相关书，然后把它们的内容加起来形成你的理解。

📐 Attention 公式三步走

$\displaystyle \text{Attention}(Q, K, V) = \text{Softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$
1

相似度评分

$Q \times K^T$。向量点积越大，方向越接近，越相关。得到一个注意力分数矩阵——每个字与其它所有字之间的相关程度。

2

归一化为权重

Softmax + $\sqrt{d_k}$ 缩放。除以 $\sqrt{d_k}$ 防止大数梯度消失。Softmax 把分数变成 0–1 的权重系数。

3

加权求和

$\text{Score} \times V$。用权重对 Value 加权求和，得到 $\Delta E$。最后加到原始向量上：$E' = E + \Delta E$。

经过这三步，初始的词向量 $E$ 不再是"白板"——它把周围相关的 $V$ 向量按比例打包在一起，当作一个"修正包"（$\Delta E$）加到了自己身上。这就是 "让每个词看见整个句子" 的数学本质。

📏 QKV 的维度解释

  • Q, K 都是降维：$d_{low} \times d_{embedding}$。从 Embedding 的高维降到低维，减少计算量同时聚焦核心特征。
  • V 的优化技巧：V 必须输出 $d_{embedding}$ 维度（否则不能加到 $E$ 上），但不一定要用 $d_{emb} \times d_{emb}$ 的大矩阵：
    → 先降维到 $d_{low}$（$d_{low} \times d_{emb}$），再升维回 $d_{emb}$（$d_{emb} \times d_{low}$）
    → 参数量从 $d_{emb}^2$ 减少为 $2 \times d_{low} \times d_{emb}$

🧠 多头注意力（Multi-Head Attention）

每个头有自己独立的 Q, K, V 矩阵，并行计算，捕捉不同的上下文关系：

  • 一个头可能关注语法关系（主谓宾）
  • 一个头可能关注指代关系（代词指代谁）
  • 一个头可能关注位置距离（附近的词）

最后，所有头的结果被拼接在一起，通过一个输出矩阵映射回 $d_{embedding}$ 维度。

🔒 掩码（Masking）

在自回归语言模型（GPT 系）中，每个 token 在预测下一个词时不能看到"未来的词"。Mask 的操作就是把未来位置的注意力分数强行设为 $-\infty$，使得 Softmax 后对应权重为 0。

这是自回归和双向模型（如 BERT）最核心的区别——一个看左边，一个看两边。

⚡ Attention 的计算量

$O(N^2)$ —— 这是大模型推理的主要瓶颈。

$N$ 是序列长度。每个 token 要和所有其他 token 计算相似度，序列翻倍，计算量翻四倍。这也是为什么长文本（如 128K 上下文）需要专门优化（如 Flash Attention）。

📌 第四幕总结

注意力机制是 Transformer 的核心创新。它通过 QKV 三矩阵的配合，实现了一个优雅的功能：让每个词能根据相关性吸收整个句子的信息。这个过程本质上是"查字典"——Q 是问题，K 是索引，V 是内容。多头注意力让模型能同时从多个维度理解上下文关系，而掩码则确保了自回归生成的方向性。

第五幕

🗃️ 模型怎么装下整个世界 — MLP 的知识存储与叠加态

Transformer 的每个基础层有两个模块：注意力模块 + 多层感知器（MLP）。多个这样的基础层堆叠而成整个模型。

注意力机制负责"谁和谁相关"，而 MLP 负责"我知道什么"。有论文指出，模型学到的事实主要存储在 MLP 中。

MLP 部分结构虽然简单，但参数量占了整个模型的2/3——它才是模型真正的知识库。

🔧 MLP 的运算过程

$\text{MLP}(x) = \sigma(x W_1 + b_1) W_2 + b_2$
1

升维 — 检索知识

$x \rightarrow \sigma(x W_1)$

用行向量视角看 $W_1$：它的每个行向量是一个"事实的 Key"（激活模式）。输入 $x$ 与每个行向量做点积，计算匹配度。

  • 升维维度一般是 $4 \times d_{embedding}$
  • 激活函数（ReLU）决定哪些知识被"点亮"
2

降维 — 组合输出

$\text{activated} \times W_2$

用列向量视角看 $W_2$：它的每一列是一个"事实的 Value"（影响方向）。前面被激活的知识，按各自强度通过 $W_2$ 的列向量组合回 Embedding 空间。

  • 最后加上残差连接：$E' \rightarrow E' + \text{MLP}(E')$
💡
直观理解：$W_1$ 的行向量是"你问我什么我就查什么"的词典索引，$W_2$ 的列向量是"查到以后在理解上添加什么方向"。整个过程就是：听懂问题 → 查出知识 → 更新理解。

🌀 叠加态（Superposition）—— 数量级的飞跃

既然模型要记住海量的世界知识，而神经元数量是有限的，模型是怎么装下这一切的？

答案来自高维空间的奇特几何性质：高维空间中可以存在大量"几乎正交"的方向。

  • 在三维空间里，最多只有 3 个互相垂直的方向（xyz 轴）
  • 但在高维空间（比如 4096 维）中，可以存在成千上万个"几乎正交"的方向
  • 模型利用这一点，通过不同的线性组合来同时编码远超神经元数量的特征和事实

这就解释了为什么大模型能成为百科全书——它并不是每个事实占一个神经元，而是叠加在一起、用不同的激活模式来区分。

⚠️ 叠加态的一个直接后果

由于叠加态的存在，你没法明确地指出某个 Key 对应哪个具体的事实——一个 Key 可能同时编码了多个知识的叠加。这就是 MLP 被称为"难以解释的模型部分"的根本原因。

📊 Transformer 参数量速查

以标准 Decoder-only Transformer（如 LLaMA）的一层为例，设 $d_{model}$ 为嵌入维度、$d_{ff} = 4 \times d_{model}$ 为 MLP 中间维度：

矩阵形状参数量作用
Embedding $W_E$$V \times d_{model}$$V \cdot d_{model}$词索引 → 高维向量
Unembedding $W_U$$d_{model} \times V$$d_{model} \cdot V$向量 → 词表分数 (Logits)
Attention Q 矩阵$d_{model} \times d_{model}$$d_{model}^2$向量 → Query（找什么）
Attention K 矩阵$d_{model} \times d_{model}$$d_{model}^2$向量 → Key（有什么）
Attention V 矩阵$d_{model} \times d_{model}$$d_{model}^2$向量 → Value（改什么）
Attention O 矩阵$d_{model} \times d_{model}$$d_{model}^2$拼接多头 → 输出
MLP Up $W_1$$d_{ff} \times d_{model}$$4 \, d_{model}^2$升维检索（Key 匹配）
MLP Down $W_2$$d_{model} \times d_{ff}$$4 \, d_{model}^2$降维组合（Value 加权）
LayerNorm (×2)$d_{model}$ (scale) + $d_{model}$ (bias)$2 \times 2 \, d_{model}$层归一化（稳定训练）
单层参数量≈ $12 \, d_{model}^2$（忽略 Embedding 和 LayerNorm），其中 MLP 占 $\frac{2}{3}$（$8d^2$）、Attention 占 $\frac{1}{3}$（$4d^2$）。整个模型的参数 ≈ 层数 × $12d^2$ + Embedding + Unembedding。

🔍 各矩阵的角色详解

  • $W_E$（Embedding）：查表。每个词对应一个固定行向量，训练时调整。参数量取决于词表大小 $V$——大词表（如 128K）是 Embedding 参数的主要来源。
  • $W_U$（Unembedding）：与 $W_E$ 可能共享权重（weight tying）以减少参数量。它从最终向量中提取每个词作为下一个词的概率依据。
  • Q / K / V 矩阵：将嵌入向量投影到 Query / Key / Value 空间。多头注意力中，$h$ 个头各有独立的 Q/K/V 矩阵，每个头维度 $d_{qkv} = d_{model} / h$，总参数量为 $3 \times d_{model}^2$。对输出的贡献通过 $W_O$ 组合回来（第 $4 \times d_{model}^2$）。
  • MLP $W_1$ 与 $W_2$：$W_1$ 的行向量是"事实 Key"，与输入做点积决定激活哪些知识；$W_2$ 的列向量是"事实 Value"，把激活的知识加权组合回嵌入空间。两者配合实现升维检索 → 降维组合的核心知识读写操作。

🔄 理解残差连接：向量不是被替换，而是被不断修正

残差连接是 Transformer 中最容易忽略却又最核心的设计之一。它的数学形式很简单：

$x_{l+1} = x_l + \text{Sublayer}(x_l)$

但它的语义含义值得深入挖掘。

🧠 直觉：皇帝 → 拿破仑

设想句子的初始词向量中包含一个词"皇帝"。在最开始的 Embedding 向量中，它只是一个孤立的词——泛指任何一位皇帝。但随着上下文逐层流过 Transformer，这个向量被不断地修正：

  • 注意力层：从句子的其他词中吸收信息（"法国"、"滑铁卢"、"1815年"），把向量朝"拿破仑"的方向推一步
  • MLP 层：基于已有的向量，从知识库中检索相关事实（"拿破仑是法国皇帝"），再修正一步
  • 下一层注意力：再吸收更细粒度的上下文……

关键点在于：残差连接不是在计算一个全新的向量，而是在原有的向量上施加修正。每一层的输出不是替换，而是"原向量 + 修正量"。

这就像一个画家在画布上不断叠加修改——最初的草图（皇帝）在每一笔之后逐渐细化（滑铁卢的皇帝 → 拿破仑）。画布本身没有被替换，上面的内容在每一层得到丰富。

📐 数学视角：为什么这是一个好设计？

  • 梯度流动：反向传播时，梯度可以直接通过残差分支回到浅层，避免了深层网络的梯度消失问题。即使层数极深，梯度也有“捷径”。
  • 信息保留：每一层只负责计算“增量”（$\Delta E$），而不需要从头构建表示。浅层的信息永远不会被深层“覆盖”——它始终存在于残差流中。
  • 层间解耦：每层只需关注“当前还需要补充什么”，而不是“重新理解整个输入”。让训练更加稳定，也使得增加层数成为一种可靠的扩展性能的方式。
📌 第五幕总结
附章二

🔀 Transformer 的向量完成哪些变换？

到目前为止，我们讨论了注意力机制（如何吸收上下文）和 MLP（如何检索知识），但还没有完整地走一遍向量在 Transformer Decoder（以 GPT 为代表的 Decoder-only 架构）中的完整路径。

一项一项来看：

步骤变换发生了什么
① Token 化文本 → 整数索引“我” → token id
② Embedding$x = W_E[t]$整数 → 稠密向量 $d_{model}$
③ 位置编码$x += PE(pos)$注入位置信息，打破置换不变性
④ 掩码自注意力$x'''= x + Attn_{masked}(x)$每个词吸收左侧所有词的信息；残差保留原向量
⑤ 层归一化$x'' = LayerNorm(x')$稳定数值范围，加速训练
⑥ MLP$x^{\prime\prime\prime}=x''+MLP(x'')$从知识库中检索并融合事实；残差保留注意力的结果
⑦ 层归一化$x^{(4)} = LayerNorm(x^{\prime\prime\prime})$再次稳定
⑧ Unembedding$logits = x^{(4)} \cdot W_U$向量 → 词表分数
⑨ Softmax$p = Softmax(logits / T)$分数 → 概率分布 → 采样下一个词
关键洞察：步骤 ④→⑤→⑥→⑦ 重复 N 层（如 LLaMA 7B 有 32 层）。每一层都在残差流上叠加修正——向量维度始终是 $d_{model}$，从未改变。改变的是向量的方向（语义），而不是空间本身。

🔻 Decoder 架构详解

Decoder-only 架构（GPT 系列）是目前大语言模型的主流。与 Encoder-Decoder 架构（原始 Transformer）不同，Decoder-only 模型只保留了解码器部分，通过因果掩码（Causal Masking）实现自回归生成。它的核心结构如下：

Image by dvgodoy / dl-visuals / CC BY 4.0

🧩
Decoder 各组件的作用：
  • Masked Self-Attention：只看到当前和之前的 token，实现自回归约束。
  • Cross-Attention（仅在 Encoder-Decoder 架构中有）：Decoder 的 Q 与 Encoder 输出的 K,V 交互。
  • Feed-Forward（MLP）：存储和检索世界知识。
  • Add & LayerNorm：残差连接 + 归一化，让深层网络在训练中保持稳定。

💡 Decoder-only vs Encoder-Decoder

  • Decoder-only（GPT）：只有 Masked Self-Attention + MLP，没有 Cross-Attention。适合自由文本生成。
  • Encoder-Decoder（T5、原始 Transformer）：Encoder 处理输入（双向注意力），Decoder 生成输出（因果注意力 + Cross-Attention）。适合翻译、摘要等 seq2seq 任务。

目前的主流大语言模型（GPT-4、Claude、LLaMA、Qwen 等）都采用 Decoder-only 架构——它更简洁，在足够的数据和规模下，性能不输 Encoder-Decoder 设计。

🎯 贯穿五条线索的主线

这份笔记的五幕 + 附章其实在讲述同一件事的多个侧面：

  • 第一幕从微观出发：每一个函数选择都是一个有意义的决定——ReLU 折纸、偏置平移、归纳偏置注入先验
  • 第二幕从宏观出发：架构决定模型能学到什么层次的规律——梯度下降只是工具，CNN 的结构先验才是它成功的本质
  • 第三幕从传播出发：反向传播是学习的引擎——链式法则把输出层的误差逐层传回，告诉你每个参数该往哪调、调多少
  • 🔧 附章一夯实数学基础：矩阵乘法的两种视角——行向量“匹配模式”、列向量“组合基底”，是所有后续机制的数学透镜
  • 第四幕进入现代架构：注意力机制让每个词看见整个句子——QKV 三矩阵配合实现上下文感知，多头并行捕捉多维关系
  • 第五幕揭示知识存储：MLP 是模型的百科全书——升维检索、降维组合、叠加态让有限神经元编码海量知识
  • 🔀 附章二完整呈现：向量从 Embedding 到输出经历的全部变换——从 Decoder 架构图到残差连接的修正直觉，再到参数量表的逐矩阵拆解，让 Transformer 这台“知识工厂”的每一个零件都可视化、可追踪。

六者交汇于同一个洞察：对数据结构和物理世界的理解，才是设计好模型的核心能力。这不仅仅是深度学习，这是"理解问题"的方法论。
