一、线性代数是为了解决什么问题？

线性代数是为了解决高维空间并发问题而诞生的数学语言。 它让我们能用高度简洁的符号体系，去操作和求解超越三维直觉的复杂系统。

人类大脑只擅长处理单变量问题——一个未知数 x，列一个方程，解一个答案。但真实世界从来不是单变量的：气象预报要处理温度、湿度、气压、风向等千万个变量；跨海大桥要计算几万个钢梁交接点的形变；短视频推荐要考虑用户历史、停留时间、点赞倾向等几百个特征。当变量成百上千时，传统代数方程彻底失效。

核心思想：无论系统里有三个变量还是三百万个变量，线性代数都可以把它们打包成一个简单的方程式：

Ax = b
  • x — 不再是单一数字，而是包含所有未知数的状态向量
  • A — 包含所有相互作用关系的规则矩阵
  • b — 已知的结果

二、几何直觉：理解线性代数的关键

要想真正学懂线性代数，你必须建立几何直觉。 不要把矩阵看成数字表格，不要把向量看成一列数字。

2.1 向量是什么？

向量 = 空间中的一个固定坐标或有向线段。它代表事物在当前空间下的一个状态。

2.2 矩阵是什么？

矩阵 = 对空间进行操作的动作。它代表一种变换规则。

矩阵乘法 Ax 的几何意义：矩阵 A 对向量 x 施加一种空间变换——可能是拉伸、旋转或剪切。

"线性"的两个死要求：

  • ✔ 原点保持固定，不能乱跑
  • ✔ 变换后的网格线依然平行，且等距分布

→ 空间不能被扭曲成麻花。你可以把变换前的空间也想象成由平行等距的网格线构成。

2.3 矩阵乘法 AB 的意义

AB = 先对空间施加 B 变换，再施加 A 变换。不是复杂的数字游戏，而是变换的复合。

2.4 秩

秩 = 变换后空间保留下来的真实维度。

2.5 常见线性变换一览

变换矩阵（2D）几何效果det
恒等[[1,0],[0,1]]什么都不变1
缩放[[k,0],[0,k]]均匀拉伸/压缩k²
拉伸[[a,0],[0,b]]各轴独立缩放a·b
旋转[[cosθ,-sinθ],[sinθ,cosθ]]绕原点旋转 θ1
反射[[1,0],[0,-1]]沿某轴翻转-1
剪切[[1,k],[0,1]]一面不动，一面平移1
投影[[1,0],[0,0]]压到一条线/一个面0
零变换[[0,0],[0,0]]压到原点0

关键规律：行列式 (det) 告诉你变换的本质：

  • det = 1 → 保持面积（旋转、剪切、恒等），不放大不缩小
  • det = -1 → 保持面积但翻转了定向（反射）
  • det = 0 → 降维了，信息丢失，其矩阵不可逆（投影、零变换）

三、几何直觉 vs 数值计算

角色作用
几何直觉解决"用什么样的工具"的问题——建模
数值计算负责求解——计算机可以直接代劳

学线性代数，重点在几何直觉（理解变换），而非数值计算（计算机替你算）。

四、向量

看待向量有两种不同的视角，取决于你是在学数学还是在编程：

物理视角

向量是有方向的线段——有大小、有方向。这是学习线性代数时应该使用的几何直觉。

计算机视角

向量是有序的数字列表——比如 [特征1, 特征2, ..., 特征n]。每个维度代表一个特征，向量的长度就是特征数。在处理计算机问题时（机器学习、数据分析），用这个视角。

4.1 多个向量的可视化

单个向量可以画成一个有向线段。但向量数量多了以后，全都画成带箭头的线会非常杂乱——此时用终点（散点）来表示每个向量，清晰得多。

五、基与线性相关

5.1 线性组合与线性空间

取两个向量 v₁, v₂，则它们的线性组合 a·v₁ + b·v₂（a, b 为任意实数）可以生成的所有向量的集合，构成一个线性空间。

5.2 基

如果一组向量能通过线性组合生成整个空间，且它们之间线性无关（谁也"不能由其他向量生成"），它们就是这个空间的一组基。

通俗理解：基就是你在一个空间中能用来"指路"的最小单位方向。每个方向都不能被其他方向替代。

增加一个新向量：如果它落在已有向量张成的空间中（线性相关），空间不变。如果它指向一个全新的方向（线性无关），空间维度就扩大了。

5.3 二维线性组合交互演示

拖动滑块调节两个系数的比例，观察向量 v = a·v₁ + b·v₂ 如何变化。

5.4 三维线性相关性交互演示

三维空间中，观察两个向量 v₁, v₂ 张成的平面。第三个向量 w 是"待在平面内"（线性相关），还是"指向新方向"（线性无关）？

六、矩阵与线性变换

不要把矩阵乘法看作是枯燥的"行乘列"公式，而要将其视为发生的几何变换。 线性变换的关键在于跟踪基向量 î 和 ĵ 的去向。

6.1 理解线性变换

  • 线性变换是将一个向量变成另外一个向量，类似于函数 y = f(x) 的变换
  • 变换后必须满足：网格线依然平行，且等距分布
  • 原点不能动

6.2 关键：跟踪基向量的去向

只要知道复合变换后基向量最终落在哪里，就能写出最终的矩阵。

对于任意向量 v = (x, y)：

v' = x·î' + y·ĵ'

假设要计算矩阵 M，只需要计算 î, ĵ 变化后的终点，分别作为 M 的第一列和第二列。

例如旋转 90°：

  • î = (1,0) → î' = (0,1) → M 第一列
  • ĵ = (0,1) → ĵ' = (-1,0) → M 第二列
  • 矩阵 M = [[0, -1], [1, 0]]

6.3 从矩阵读回变换

拿到一个矩阵 M = [[1, k], [0, 1]]，查看 î' 和 ĵ' 就能知道空间发生了什么——这就是剪切（Shear）。拖动滑块观察：变换后的网格依然平行、等距。

6.4 矩阵×向量：列组合视角

拖动角度滑块或拖拽向量 v 的顶点，观察基向量的去向如何构成矩阵，以及 M·v = x·î' + y·ĵ' 的计算过程。

七、矩阵乘法与线性变换复合关系

A·B 有两种几何视角：列组合视角（每列分别变换）、变换复合视角（先 B 再 A）。

7.1 列组合视角

A·B = [A·b₁ A·b₂ ... A·bₚ]

把 B 的每一列看作一个向量，A·B 就是对每个列向量分别应用变换 A。

结果矩阵的第 j 列 = A·bⱼ = (b₁ⱼ)·col₁(A) + (b₂ⱼ)·col₂(A)。这就是"列组合"——用 B 的列元素作为系数，组合 A 的列。

7.2 变换复合视角

AB = 先对空间施加 B 变换，再施加 A 变换。这是更常见的视角，后续的很多概念也围绕这个思想展开。

这类似于复合函数 f(g(x))——从右往左读：

(A·B)·v = A·(B·v)

先对 v 做 B 变换，再对结果做 A 变换。

7.3 复合变换交互演示

切换 A·B / B·A 两种顺序，用步骤按钮观察每一步的变化。同样的旋转和剪切，顺序不同 → 结果不同 → 直观理解为什么矩阵乘法不满足交换律。

7.4 结合律交互演示

三个变换按相同顺序（C → B → A）施加到空间，无论怎么加括号，最终结果都相同。因为矩阵乘法对应函数复合，而函数复合天然满足结合律。

几何视角：

  • (A·B)·C — 先对向量施加 C 变换，然后将结果施加 (A·B) 变换。而 (A·B) 是"先 B 再 A"的复合，所以整体 = C → B → A。
  • A·(B·C) — 先对向量施加 (B·C) 变换（即"先 C 再 B"），再对结果施加 A 变换。整体 = C → B → A。

两者都表示"先 C、再 B、再 A"的同一串连续变换——括号只是标明了先计算哪两个矩阵的乘积，没有改变变换的顺序和内容，所以必然相等：

(A·B)·C = A·(B·C) = 先 C 再 B 再 A

7.5 使用复合线性变换的角度推导矩阵乘法的计算方法

八、三维空间

以上二维空间的变换和几何视角，同样适用于三维空间。基向量从 î, ĵ 变为 î, ĵ, k̂，矩阵从 2×2 变为 3×3，但核心思想完全一致：

  • 矩阵的每一列 = 变换后的基向量
  • 矩阵乘法 = 复合变换
  • 行列式 = 体积缩放因子
  • 秩 = 变换后空间的维度

九、行列式

行列式描述了一个线性变换对空间拉伸或压缩的尺度。 对二维来说就是面积缩放因子，对三维来说就是体积缩放因子。

9.1 面积缩放因子

在做线性变换时，会对空间进行拉伸或挤压。关键特性：无论选哪个区域的面积，其缩放比例都是相同的（均匀缩放）。

具体例子：

  • 行列式的值为 3 → 对应矩阵变换将任意面积变成原来的 3 倍
  • 行列式为 0 → 面积变成 0，说明变换将二维平面压缩成一条线或一个点（降维，矩阵不可逆）

9.2 行列式的负值

负值的含义——空间定向的翻转（Orientation Flip）。

如果把二维平面看成一张纸，行列式为负相当于变换把纸翻了一个面。

以基向量 î, ĵ 判断：如果变换后 ĵ 相对于 î 的相对左右关系反了（原本在左侧，变换后到了右侧），行列式就是负值。

行列式的绝对值依然表示缩放比例，负号仅代表方向翻转。

9.3 三维空间的行列式

三维行列式 = 体积的缩放因子。负值在三维中代表手性翻转（左手系 ↔ 右手系）。

关键规律总结：

  • |det| > 1 → 空间被放大
  • |det| = 1 → 面积/体积不变（旋转、剪切）
  • 0 < |det| < 1 → 空间被压缩
  • det = 0 → 降维了，信息丢失，矩阵不可逆
  • det < 0 → 空间定向翻转，绝对值仍表示缩放比例

十、非方阵

前面讨论的都是方阵（二维→二维，三维→三维）。非方阵的几何含义是什么？

10.1 3行2列的矩阵（2D → 3D）

  • 矩阵有 2列，代表有 2个基向量（原始空间是二维）
  • 矩阵有 3行，表示每个基向量都有 3个独立坐标来描述
  • 这是 二维空间到三维空间的变换
  • 举例：将二维平面上的点 (x, y) 映射到三维空间中的 (ax + by, cx + dy, ex + fy)

10.2 2行3列的矩阵（3D → 2D）

  • 矩阵有 3列，代表有 3个基向量（原始空间是三维）
  • 矩阵有 2行，表示每个基向量都有 2个独立坐标来描述（变换后由2个坐标描述）
  • 这是 三维空间到二维空间的变换
  • 举例：将三维空间中的点 (x, y, z) 投影到二维平面的 (ax + by + cz, dx + ey + fz)

10.3 理解 A·x 的几何意义（线性变换的导航视角）

变换前：向量 x = (x₁, x₂) 是一组导航指令——沿着标准基 î 走 x₁ 步，再沿着 ĵ 走 x₂ 步，到达终点。

变换 A：将基向量 î, ĵ 移动到新的位置 î', ĵ'（即矩阵 A 的第一列和第二列）。

变换后：同样的系数 x₁, x₂ 变成沿着新基向量 î', ĵ' 导航——x₁ 步沿 î'，x₂ 步沿 ĵ'，到达终点 A·x。

核心直觉：x 的分量没变，变的是它们导航所依赖的"地图"（基）。

注意：输出用几个坐标描述（行数），不等于有效维度（秩）。例如 3行2列矩阵的列向量可能只张成一个平面，秩=2 < 行数=3，"空维度"永远到不了。

关键规律：矩阵的列数 = 原始空间的维度（基向量个数），行数 = 变换后空间的维度（每个基向量的坐标数）。
