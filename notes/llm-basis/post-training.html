如果说预训练是让模型在大海里"读书破万卷"，那么后训练就是"社会化规范训练"——把一个满腹经纶但满嘴胡话的"野生"模型，驯化成一个听话、有用、安全的"AI 助手"。

SFT 和 RLHF 是大模型后训练阶段最核心的两个方法。

SFT（监督微调）

用高质量、人工编写的"提示词—完美答案"对来训练模型。

{"prompt": "请把这段话翻译成英文：...", "response": "..."}
  • 核心作用：教模型学会"格式"与"听懂指令"
  • SFT 强行扭转模型的行为模式，教它看到问号就要给出答案，看到"翻译"就要执行翻译
  • 把它从一个"续写软件"变成一个"对话助手"

RLHF（基于人类反馈的强化学习）

SFT 结束后模型已会听懂人话，但仍可能狡辩、敷衍或给出有害回答。RLHF 引入人类偏好，通过强化学习来规范模型。

训练奖励模型（Reward Model）

  • 让人类给模型的多个回答打分（回答A礼貌且准确 5 分，回答B态度恶劣 1 分）
  • 让一个单独的模型学会"人类喜欢什么样的回答"

PPO/DPO 强化学习

让大模型自己去回答问题，如果它的回答符合人类偏好，奖励模型就给它加分，否则扣分。

Post Training vs. Agent Engineer

后训练工程师

  • 核心目标：让模型"懂代码"、"会思考"、"听指令"
  • 工作是在模型出厂前，通过数据喂养来改变它的"神经元"
  • 构造思维链数据、代码质量对齐、错误纠正能力、偏好学习 (DPO)
  • 交付物：一个 .pth 或 .safetensors 模型权重文件

Agent Engineer

  • 核心目标：让模型"能运行"、"会查文档"、"能自主闭环"
  • 把那个"大脑"放进一个真实的开发环境中
  • 环境隔离、工具编排、长程规划与状态管理、记忆检索 (RAG)
  • 交付物：一个复杂的软件系统，能打开 VS Code，自己跑测试，遇到报错会翻文档，最后提交 PR

对比总结

维度 后训练工程师 (Model Side) Agent 工程师 (System Side)
核心目标 提高模型基础能力 (如 Function Calling) 提高任务完成率 (Success Rate)
主要工具 PyTorch, DeepSpeed, Llama-Factory LangGraph, Python, Vector DB
关注点 损失函数 (Loss)、困惑度 (PPL) 逻辑闭环、延迟、API 稳定性
交付物 一个 Checkpoint (模型权重) 一个可运行的业务系统 (Service)
💡 总结

SFT 教模型"听懂指令"，RLHF 教模型"符合人类偏好"。后训练工程师优化模型的"内在能力"，Agent 工程师构建模型的"外部系统"。
