第一幕

🧠 神经网络的函数选择 — 每个变换都在编码一个决定

神经网络是一系列函数（变换）的组合。但选择什么函数从来不是任意的——每个激活函数、每个规范化操作，都在编码一个设计者对于"这个世界是什么样的"的假设。

选择函数，就是在选择满足特定性质的变换。这些性质可以归为两个维度。

1

数学运算的要求

—— 约束来自"可行"

  • 输出必须能解释为概率（softmax）
  • 梯度不能爆炸 / 消失（求导链式法则的约束）
  • 数值稳定性、尺度不变性、零中心化

这些是必要条件。不满足，网络就训练不好。

2

数学上的意义

—— 追问"为什么这个函数能代表现实规律？"

  • 拓扑学意义：ReLU 折纸般折叠高维空间
  • 统计学意义：sigmoid 模拟神经元激活概率
  • 先验知识：CNN 注入"局部相关性"假设

这些是深刻的设计哲学。理解了它，你就能理解为什么有些架构天然更好。

🏔️ 深潜一：ReLU 为什么更好？—— 一段函数筛选史

今天我们知道 ReLU 比 sigmoid 好，但当初并不是这样推理出来的：

  1. 早期（数学驱动）：sigmoid 的灵感来自生物神经元的"激活阈值"——看起来有生物学依据，数学上也光滑可导，所以被广泛使用
  2. 中期（工程试错）：人们发现深层网络用 sigmoid 训练不动了——梯度在链式法则下一层层地消失。ReLU 的常数导数（x > 0 时导数为 1）完美解决了这个问题，但当时并不完全清楚为什么它"这么好"
  3. 后期（理论解释）：数学和优化理论给出了优雅的解释——ReLU 对高维空间的"折纸"操作为什么从根本上改变了数据分布的拓扑结构
启示：选择激活函数，绝大部分时候就是在选择它的导数。基于梯度下降的神经网络中，导数特性决定了训练成败。

🔬 ReLU 的拓扑学意义

ReLU 在数学上相当于对高维空间进行弯折（folding / creasing）。

每一个 ReLU 神经元都像是一把"剪刀"，把空间沿着某条直线折叠。成千上万个 ReLU 组合起来，就能把原本线性不可分的高维空间，像折纸一样层层折叠、裁剪，最终让不同类别的数据在新的空间里变得线性可分。

这不仅仅是一个工程技巧——这是在用几何变换重新组织信息的表达方式。

⚖️ 深潜二：偏置（Bias）—— 一个"小参数"背后的几何革命

💡
核心直觉：如果只有 $W\vec{x}$，原点 $(0,0,...,0)$ 永远被固定死。无论怎么旋转、拉伸，原点变换后依然是原点（$W \cdot \vec{0} = \vec{0}$）。但在现实任务中，分类边界几乎从不经过原点。偏置 $\vec{b}$ 的唯一作用就是平移整个高维空间，赋予网络脱离原点束缚的能力。

🤔 偏置能用矩阵替代吗？

可以——通过齐次坐标（Homogeneous Coordinates）给向量升一维，把平移融入矩阵乘法。

但深度学习框架仍然显式写 +b：

  • 显式表达：权重（敏感度）和偏置（基准线）分开，更符合直觉
  • 节省内存：强行拼接 1 到数据中徒增存储，广播机制在 GPU 上更高效

🔗 偏置的神经网络解释

在 sigmoid 时代，偏置直接决定了神经元是否容易被激活——当输入加权和超过偏置阈值时，神经元才"兴奋"。

偏置越大，神经元越难激活（阈值更高）。这也是早期神经科学启发的一个设计痕迹。

· · ·

🏗️ 不把网络当黑盒

理解这些"为什么"的实用价值：当模型出问题时，它指导你从哪个方向优化。

反例：梯度消失 → 你知道该检查激活函数和初始化方式，而不是盲目调学习率。

正例：CNN 包含的先验知识（局部性 + 平移不变性）让它能以数量级更少的数据达到相同的效果。

📌 第一幕总结

神经网络的函数选择不是随意的——每个变换都在编码一个关于世界的假设。数学可行性和哲学理解是两条并行的线索，交汇在一起决定了"什么架构能 work"。理解这些，是走出"调参工程师"的第一步。

第二幕

📐 梯度下降 — 是怎么学的，和为什么学不会是两回事

🎬 3Blue1Brown — 梯度下降（视频）

梯度下降是神经网络学习的机制，但它不保证学到有意义的东西。接下来的实验会暴露这个深刻的区别。

⚙️ 机制：梯度下降简述

  • 单变量函数 → 求导找最小值
  • 多变量函数 → 梯度是一个向量，指向"最陡上升"方向，取反就是下山方向
  • loss function 定义"错误程度" → 梯度告诉你如何调整参数来减小错误
⚠️ 这是多变量微积分的内容，如果三维梯度的几何意义不清晰，值得停下来补充一下。这是所有后续理解的基础。
↓

❌ 实验：两层的数字识别网络 — 它没有按预期工作

😤 期望 vs 现实
🟢 我们期望的

第一层识别"横、竖、撇、捺"等笔画部件，第二层组合这些部件来识别数字。层级结构**有意义的**抽象。

🔴 实际发生的

用随机图片测试，网络给出错误输出。特征可视化显示第一层看到的不是笔画，而是噪音般的模糊鬼影——它根本没学会我们想象中的"特征"。

梯度下降在运行，loss 在下降，但网络学到的不是我们想要的东西。这就是机制正确但意义缺失。

↓ 为什么？三个根本原因 ⟶

🔍 诊断：三个原因解释为什么"学好"不等于"学会"

1

全连接的"全局性"摧毁了空间局部性

——每个神经元看整张图

  • 现实：每个隐藏神经元与所有像素相连
  • 后果：神经元不关注"局部"，而是全局统计。最终学到的不是笔画边缘，而是斑驳光点

→ 教训：没有局部性假设，模型只能找粗糙的统计规律

2

浅层网络（仅两层）缺乏"抽象深度"

——像素→边缘→纹理→部件→整体的层次需要多层

  • 两层网络没有足够层级来实现分层抽象
  • 它"走捷径"：直接拟合像素→标签的非线性映射，而不是构建有意义的中间表示

→ 教训：深度不是为了"更深的网络听起来更厉害"，而是层次结构是抽象的必要条件

3

缺乏"平移不变性"

——"7"向右移 2px，对网络来说就是全新输入

  • 全连接网络对位置极其敏感：换个位置，之前学到的权重完全对不上
  • 没有不变的图像，只有无穷多的变体

→ 教训：没有不变性假设，需要指数级数据来覆盖所有变体

💊 解法：CNN 如何用"结构先验"同时解决这三个问题

CNN 之所以大幅超越全连接网络，不是因为它"更聪明"，而是因为它在设计之初就注入了关于图像世界的物理假设：

🏋️ 权重共享

同一个卷积核在整个图像上滑动，参数只有一小套。这不仅减少了参数量，更关键的是——无论特征出现在画面的哪个位置，都能被同一组参数检测到。

🧠 归纳偏置（Inductive Bias）

模型在训练前就被注入了两大假设：

  • 局部相关性——靠得近的像素关系更密切
  • 平移不变性——同一物体换个位置还是同一物体
CNN 在训练开始之前，已经站在了终点线附近。
🔗
这里完美呼应了 how2ai-course 的内容：找到数据的不变性能大幅减少所需样本量。CNN 找到的不变性是"平移"——所以它不需要像 MLP 那样，用成千上万张"往右平移 5 像素"的图片去肉搏学习这个规律。这不仅仅是效率提升，这是学习范式的根本转变。
📌 第二幕总结

梯度下降是学习的引擎，但架构才是学习的方向盘。

两层的全连接网络在数字识别上的失败揭示了一条核心规律：没有结构先验的模型，需要用指数级的数据和参数来弥补架构的盲目性。CNN 的成功不是因为它"更深"或"更新"，而是因为它编码了对这个物理世界的基本假设——而这些假设，正是我们通过"理解数据、找到不变性"得到的洞察。

第三幕

🔙 反向传播（Back Propagation）— 学习的"幕后推手"

🎬 3B1B — 反向传播（直觉篇） 🎬 3B1B — 反向传播（微积分篇） 📺 B站 — 反向传播微积分

所谓的学习，就是找到所有参数组合，使得 loss function 最小。

对于手写识别神经网络，我们需要求 13000 个参数的负梯度。虽然参数分布在不同的网络层次上，但我们可以先把它们简单地看作一个梯度——每个参数对 loss function 的敏感程度不一样。

举个例子：如果一个权重是 3.2，另一个是 0.1，则第一个有扰动时对 loss 的影响大约是第二个的 32 倍。敏感度差异巨大。

↓ 让我们通过"数字 2"的故事来理解反向传播的直觉 ⟶

👁️ 直觉：识别数字"2"背后的三个动作

在输出层，网络要对"这张图是不是 2"做出判断。反向传播会依次做三件事，每一件都有清晰的直觉逻辑：

1

改变 Bias

直接给"2"的输出神经元加温

Bias 就像一个神经元的"基础门槛"或"自带分值"。如果不管输入是什么，我都想让"2"的概率变大，最直接的方法就是把"2"这个神经元的 Bias 调大——相当于降低激活"2"的门槛。

2

改变 Weight

顺藤摸瓜，给贡献大的前层神经元"发奖金"

"2"的输出是由倒数第二层的神经元乘以各自的权重相加得到的。如果倒数第二层某个神经元 $A$ 正处于高激活状态（比如它捕捉到了"2"底部的横线），把 $A$ 连接到"2"的权重调大，就能极大地放大这个正向刺激。

"大的权重调整起来更有性价比"——因为权重调整幅度正比于前层激活值。前层越兴奋，调整它的权重效果越立竿见影。
3

改变前一层激活值

对前一层提要求——"有功的再兴奋一点，帮倒忙的闭嘴"

光改当前参数（Bias 和 Weight）是有上限的。反向传播最厉害的地方在于，它会向倒数第二层传达一个愿望（Desire）：

  • 对连接着正权重的神经元 → "再亮一点"（激活值变大）
  • 对连接着负权重的神经元 → "再暗一点"（激活值变小）
💡
贯穿逻辑：这三步本质上是在回答一个问题——"如果要降低 loss，每个参数应该往哪个方向调整多少？" Bias 直接调门槛，Weight 放大关键信号，激活值调整则为下一轮的反向传播做好了准备。每一层都做同样的事情，信号就从输出层一层层传回了输入层。

⚡ mini-batch 与 SGD

实际训练中，不会只用一个样本的梯度来更新参数——那样太毛糙（每个样本的"愿望"都不同）。

相反，用一小批（mini-batch）样本的平均梯度来决定更新方向。这就是随机梯度下降（SGD）——在效率和稳定性之间取平衡。

🔬 反向传播的微积分 — 从扰动到链式法则

直觉给了我们方向，但微积分给了我们精确的数字。

反向传播本质上就是微分的链式法则在神经网络中的应用。以下用"多条数轴联动"和"微小扰动（Nudges）"的直观方式来解释。

📐 核心直觉：导数 = 对局部扰动的敏感度

三条基本认识

  • $\frac{dy}{dh}$：如果系统在 $h$ 这个位置被轻轻拨动了 $dh$，最终的输出 $y$ 会产生多大的波动？—— 这是 $y$ 对 $h$ 的直接敏感度。
  • $\frac{dy}{dx}$：如果源头 $x$ 被拨动了 $dx$，经过中间所有多米诺骨牌的连锁反应后，最终的 $y$ 会产生多大的波动？—— 这是 $y$ 对 $x$ 的全路径敏感度。
  • 为什么需要"不同的中间增量"来表达？ 因为在实际面对复杂系统时，我们往往无法一眼看穿最底层（$x$）和最顶层（$y$）的关系，但我们能看清相邻变动的关系。链式法则就是把这些"相邻片段"串起来。
🧩
在神经网络中的映射：$x$ 是输入的图像 → 中间隐藏神经元是无数 $h_1, h_2, ...$ → $y$ 是最终的预测误差 Loss。为了调整网络，我们需要知道"每一个中间的神经元如果产生扰动，对最终的 Loss 会有什么影响"——这正是链式法则解决的问题。

🧮 极简网络举例

从"每层只有一个神经元"的极简网络开始：

$a^{(L-1)} \xrightarrow{\times w^{(L)} + b^{(L)}} z^{(L)} \xrightarrow{\sigma(z)} a^{(L)} \xrightarrow{\text{计算与 } y \text{ 的差距}} C_0$
  • $a^{(L-1)}$：前一层神经元的激活值
  • $z^{(L)} = w^{(L)} a^{(L-1)} + b^{(L)}$：当前的加权输入（Weighted sum）
  • $a^{(L)} = \sigma(z^{(L)})$：当前层的激活值（$\sigma$ 为激活函数，如 Sigmoid 或 ReLU）
  • $C_0 = (a^{(L)} - y)^2$：单个样本的代价（$y$ 是标签，即期望输出值）

扰动传导链

如果我们对 $w^{(L)}$ 微调一个极其微小的量 $\partial w^{(L)}$，这个扰动会像水波一样向右传导：

$\partial w^{(L)} \rightarrow \partial z^{(L)} \rightarrow \partial a^{(L)} \rightarrow \partial C_0$

根据链式法则，代价 $C_0$ 对权重 $w^{(L)}$ 的偏导数（敏感度）就是这三级跳的乘积：

$\displaystyle \frac{\partial C_0}{\partial w^{(L)}} = \frac{\partial C_0}{\partial a^{(L)}} \cdot \frac{\partial a^{(L)}}{\partial z^{(L)}} \cdot \frac{\partial z^{(L)}}{\partial w^{(L)}}$

推导最后一层到前一层，和前一层到更前一层是类似的——递归地应用同样的逻辑。

复杂网络的推导跟极简网络也是类似的——只是多写了下标。
📌 第三幕总结

反向传播是神经网络学习的引擎：它用微分的链式法则，把输出层的误差信号逐层反向传播到每一个参数，告诉我们"每个权重该往哪个方向调整多少"。直觉层面的三个动作（调 Bias、调 Weight、调前层激活值）和微积分层面的链式法则是同一件事的两种语言——一个让你理解"为什么"，一个让你算出"是多少"。

🎯 贯穿三条线索的主线

这份笔记的三幕其实在讲述同一件事的三个侧面：

  • 第一幕从微观出发：每一个函数选择都是一个有意义的决定——ReLU 折纸、偏置平移、归纳偏置注入先验
  • 第二幕从宏观出发：架构决定模型能学到什么层次的规律——梯度下降只是工具，CNN 的结构先验才是它成功的本质
  • 第三幕从传播出发：反向传播是学习的引擎——链式法则把输出层的误差逐层传回，告诉你每个参数该往哪调、调多少

三者交汇于同一个洞察：对数据结构和物理世界的理解，才是设计好模型的核心能力。这不仅仅是深度学习，这是"理解问题"的方法论。
