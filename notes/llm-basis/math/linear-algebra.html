一、线性代数是为了解决什么问题？

线性代数是为了解决高维空间并发问题而诞生的数学语言。 它让我们能用高度简洁的符号体系，去操作和求解超越三维直觉的复杂系统。

人类大脑只擅长处理单变量问题——一个未知数 x，列一个方程，解一个答案。但真实世界从来不是单变量的：气象预报要处理温度、湿度、气压、风向等千万个变量；跨海大桥要计算几万个钢梁交接点的形变；短视频推荐要考虑用户历史、停留时间、点赞倾向等几百个特征。当变量成百上千时，传统代数方程彻底失效。

核心思想：无论系统里有三个变量还是三百万个变量，线性代数都可以把它们打包成一个简单的方程式：

Ax = b
  • x — 不再是单一数字，而是包含所有未知数的状态向量
  • A — 包含所有相互作用关系的规则矩阵
  • b — 已知的结果

二、几何直觉：理解线性代数的关键

要想真正学懂线性代数，你必须建立几何直觉。 不要把矩阵看成数字表格，不要把向量看成一列数字。

2.1 向量是什么？

向量 = 空间中的一个固定坐标或有向线段。它代表事物在当前空间下的一个状态。

2.2 矩阵是什么？

矩阵 = 对空间进行操作的动作。它代表一种变换规则。

矩阵乘法 Ax 的几何意义：矩阵 A 对向量 x 施加一种空间变换——可能是拉伸、旋转或剪切。

"线性"的两个死要求：

  • ✔ 原点保持固定，不能乱跑
  • ✔ 变换后的网格线依然平行，且等距分布

→ 空间不能被扭曲成麻花。你可以把变换前的空间也想象成由平行等距的网格线构成。

2.3 矩阵乘法 AB 的意义

AB = 先对空间施加 B 变换，再施加 A 变换。不是复杂的数字游戏，而是变换的复合。

2.4 秩

秩 = 变换后空间保留下来的真实维度。

2.5 常见线性变换一览

变换矩阵（2D）几何效果det
恒等[[1,0],[0,1]]什么都不变1
缩放[[k,0],[0,k]]均匀拉伸/压缩k²
拉伸[[a,0],[0,b]]各轴独立缩放a·b
旋转[[cosθ,-sinθ],[sinθ,cosθ]]绕原点旋转 θ1
反射[[1,0],[0,-1]]沿某轴翻转-1
剪切[[1,k],[0,1]]一面不动，一面平移1
投影[[1,0],[0,0]]压到一条线/一个面0
零变换[[0,0],[0,0]]压到原点0

关键规律：行列式 (det) 告诉你变换的本质：

  • det = 1 → 保持面积（旋转、剪切、恒等），不放大不缩小
  • det = -1 → 保持面积但翻转了定向（反射）
  • det = 0 → 降维了，信息丢失，其矩阵不可逆（投影、零变换）

三、几何直觉 vs 数值计算

角色作用
几何直觉解决"用什么样的工具"的问题——建模
数值计算负责求解——计算机可以直接代劳

学线性代数，重点在几何直觉（理解变换），而非数值计算（计算机替你算）。

四、向量

看待向量有两种不同的视角，取决于你是在学数学还是在编程：

物理视角

向量是有方向的线段——有大小、有方向。这是学习线性代数时应该使用的几何直觉。

计算机视角

向量是有序的数字列表——比如 [特征1, 特征2, ..., 特征n]。每个维度代表一个特征，向量的长度就是特征数。在处理计算机问题时（机器学习、数据分析），用这个视角。

4.1 多个向量的可视化

单个向量可以画成一个有向线段。但向量数量多了以后，全都画成带箭头的线会非常杂乱——此时用终点（散点）来表示每个向量，清晰得多。

五、基与线性相关

5.1 线性组合与线性空间

取两个向量 v₁, v₂，则它们的线性组合 a·v₁ + b·v₂（a, b 为任意实数）可以生成的所有向量的集合，构成一个线性空间。

5.2 基

如果一组向量能通过线性组合生成整个空间，且它们之间线性无关（谁也"不能由其他向量生成"），它们就是这个空间的一组基。

通俗理解：基就是你在一个空间中能用来"指路"的最小单位方向。每个方向都不能被其他方向替代。

增加一个新向量：如果它落在已有向量张成的空间中（线性相关），空间不变。如果它指向一个全新的方向（线性无关），空间维度就扩大了。

5.3 二维线性组合交互演示

拖动滑块调节两个系数的比例，观察向量 v = a·v₁ + b·v₂ 如何变化。

5.4 三维线性相关性交互演示

三维空间中，观察两个向量 v₁, v₂ 张成的平面。第三个向量 w 是"待在平面内"（线性相关），还是"指向新方向"（线性无关）？

六、矩阵与线性变换

不要把矩阵乘法看作是枯燥的"行乘列"公式，而要将其视为发生的几何变换。 线性变换的关键在于跟踪基向量 î 和 ĵ 的去向。

6.1 理解线性变换

  • 线性变换是将一个向量变成另外一个向量，类似于函数 y = f(x) 的变换
  • 变换后必须满足：网格线依然平行，且等距分布
  • 原点不能动

6.2 关键：跟踪基向量的去向

只要知道复合变换后基向量最终落在哪里，就能写出最终的矩阵。

对于任意向量 v = (x, y)：

v' = x·î' + y·ĵ'

假设要计算矩阵 M，只需要计算 î, ĵ 变化后的终点，分别作为 M 的第一列和第二列。

例如旋转 90°：

  • î = (1,0) → î' = (0,1) → M 第一列
  • ĵ = (0,1) → ĵ' = (-1,0) → M 第二列
  • 矩阵 M = [[0, -1], [1, 0]]

6.3 从矩阵读回变换

拿到一个矩阵 M = [[1, k], [0, 1]]，查看 î' 和 ĵ' 就能知道空间发生了什么——这就是剪切（Shear）。拖动滑块观察：变换后的网格依然平行、等距。

6.4 矩阵×向量：列组合视角

拖动角度滑块或拖拽向量 v 的顶点，观察基向量的去向如何构成矩阵，以及 M·v = x·î' + y·ĵ' 的计算过程。

七、矩阵乘法与线性变换复合关系

A·B 有两种几何视角：列组合视角（每列分别变换）、变换复合视角（先 B 再 A）。

7.1 列组合视角

A·B = [A·b₁ A·b₂ ... A·bₚ]

把 B 的每一列看作一个向量，A·B 就是对每个列向量分别应用变换 A。

结果矩阵的第 j 列 = A·bⱼ = (b₁ⱼ)·col₁(A) + (b₂ⱼ)·col₂(A)。这就是"列组合"——用 B 的列元素作为系数，组合 A 的列。

7.2 变换复合视角

AB = 先对空间施加 B 变换，再施加 A 变换。这是更常见的视角，后续的很多概念也围绕这个思想展开。

这类似于复合函数 f(g(x))——从右往左读：

(A·B)·v = A·(B·v)

先对 v 做 B 变换，再对结果做 A 变换。

7.3 复合变换交互演示

切换 A·B / B·A 两种顺序，用步骤按钮观察每一步的变化。同样的旋转和剪切，顺序不同 → 结果不同 → 直观理解为什么矩阵乘法不满足交换律。

7.4 结合律交互演示

三个变换按相同顺序（C → B → A）施加到空间，无论怎么加括号，最终结果都相同。因为矩阵乘法对应函数复合，而函数复合天然满足结合律。

几何视角：

  • (A·B)·C — 先对向量施加 C 变换，然后将结果施加 (A·B) 变换。而 (A·B) 是"先 B 再 A"的复合，所以整体 = C → B → A。
  • A·(B·C) — 先对向量施加 (B·C) 变换（即"先 C 再 B"），再对结果施加 A 变换。整体 = C → B → A。

两者都表示"先 C、再 B、再 A"的同一串连续变换——括号只是标明了先计算哪两个矩阵的乘积，没有改变变换的顺序和内容，所以必然相等：

(A·B)·C = A·(B·C) = 先 C 再 B 再 A

7.5 使用复合线性变换的角度推导矩阵乘法的计算方法

八、三维空间

以上二维空间的变换和几何视角，同样适用于三维空间。基向量从 î, ĵ 变为 î, ĵ, k̂，矩阵从 2×2 变为 3×3，但核心思想完全一致：

  • 矩阵的每一列 = 变换后的基向量
  • 矩阵乘法 = 复合变换
  • 行列式 = 体积缩放因子
  • 秩 = 变换后空间的维度

九、行列式

行列式描述了一个线性变换对空间拉伸或压缩的尺度。 对二维来说就是面积缩放因子，对三维来说就是体积缩放因子。

9.1 面积缩放因子

在做线性变换时，会对空间进行拉伸或挤压。关键特性：无论选哪个区域的面积，其缩放比例都是相同的（均匀缩放）。

具体例子：

  • 行列式的值为 3 → 对应矩阵变换将任意面积变成原来的 3 倍
  • 行列式为 0 → 面积变成 0，说明变换将二维平面压缩成一条线或一个点（降维，矩阵不可逆）

9.2 行列式的负值

负值的含义——空间定向的翻转（Orientation Flip）。

如果把二维平面看成一张纸，行列式为负相当于变换把纸翻了一个面。

以基向量 î, ĵ 判断：如果变换后 ĵ 相对于 î 的相对左右关系反了（原本在左侧，变换后到了右侧），行列式就是负值。

行列式的绝对值依然表示缩放比例，负号仅代表方向翻转。

9.3 三维空间的行列式

三维行列式 = 体积的缩放因子。负值在三维中代表手性翻转（左手系 ↔ 右手系）。

关键规律总结：

  • |det| > 1 → 空间被放大
  • |det| = 1 → 面积/体积不变（旋转、剪切）
  • 0 < |det| < 1 → 空间被压缩
  • det = 0 → 降维了，信息丢失，矩阵不可逆
  • det < 0 → 空间定向翻转，绝对值仍表示缩放比例

十、非方阵

前面讨论的都是方阵（二维→二维，三维→三维）。非方阵的几何含义是什么？

10.1 3行2列的矩阵（2D → 3D）

  • 矩阵有 2列，代表有 2个基向量（原始空间是二维）
  • 矩阵有 3行，表示每个基向量都有 3个独立坐标来描述
  • 这是 二维空间到三维空间的变换
  • 举例：将二维平面上的点 (x, y) 映射到三维空间中的 (ax + by, cx + dy, ex + fy)

10.2 2行3列的矩阵（3D → 2D）

  • 矩阵有 3列，代表有 3个基向量（原始空间是三维）
  • 矩阵有 2行，表示每个基向量都有 2个独立坐标来描述（变换后由2个坐标描述）
  • 这是 三维空间到二维空间的变换
  • 举例：将三维空间中的点 (x, y, z) 投影到二维平面的 (ax + by + cz, dx + ey + fz)

10.3 理解 A·x 的几何意义（线性变换的导航视角）

变换前：向量 x = (x₁, x₂) 是一组导航指令——沿着标准基 î 走 x₁ 步，再沿着 ĵ 走 x₂ 步，到达终点。

变换 A：将基向量 î, ĵ 移动到新的位置 î', ĵ'（即矩阵 A 的第一列和第二列）。

变换后：同样的系数 x₁, x₂ 变成沿着新基向量 î', ĵ' 导航——x₁ 步沿 î'，x₂ 步沿 ĵ'，到达终点 A·x。

核心直觉：x 的分量没变，变的是它们导航所依赖的"地图"（基）。

注意：输出用几个坐标描述（行数），不等于有效维度（秩）。例如 3行2列矩阵的列向量可能只张成一个平面，秩=2 < 行数=3，"空维度"永远到不了。

关键规律：矩阵的列数 = 原始空间的维度（基向量个数），行数 = 变换后空间的维度（每个基向量的坐标数）。

十一、点积与对偶性

点积有两种等价的视角：投影几何与线性变换。 前者直观，后者则揭示了点积与线性代数核心概念的深刻联系——对偶性。

11.1 投影几何视角

v · w = ||v|| · ||w|| · cos(θ)

  • 将向量 v 垂直投影到向量 w 的方向上（或者反过来）
  • 将投影得到的长度，乘以 w 本身的长度

11.2 线性变换视角（对偶性）

假设有一个线性变换 f，它把一个高维空间中的向量映射到一维数轴上。也就是说，这个变换的输入是一个向量，输出是一个普通的数字（标量）。

  • 线性变换的核心性质：保持空间的网格平行且等距。当一个高维空间被"压扁"到一条一维数轴上时，空间中原本等距的网格线，在一维数轴上依然会映射成等距的点。
  • 既然 f 是一个线性变换，我们只需要看它对标准基（î 和 ĵ）做了什么，就能完全确定这个变换。

11.3 关键推导

假设线性变换 f 把 î 映射到了数轴上的数字 a，把 ĵ 映射到了数字 b。那么，对于任意向量 v = (x, y)：

f(v) = x · f(î) + y · f(ĵ) = ax + by

细心观察这个结果：ax + by。这不就是向量 (a, b) 和向量 (x, y) 的点积吗？

核心洞察：当你计算两个向量 w · v 的点积时，在几何上，你可以将 w 视为一个将高维空间压扁到一维的线性变换。点积的结果，就是向量 v 在这个变换（投影并根据 w 的长度进行缩放）之下的输出值。

11.4 与矩阵的统一

这个视角把点积的运算也看成了线性变换——与前面"矩阵=变换"的主线完全自洽：

  • 向量 w 对应的线性变换，其矩阵就是 w 的转置（1 × n 矩阵）
  • w · v = wT · v —— 一个 1 × n 的矩阵乘以一个 n × 1 的向量（矩阵乘法）
  • 所有的数学运算（乘法、求逆、点积）都统一到了"变换"的几何视角下

十二、基变换

基变换就是用不同的"镜头"（坐标系）来观察同一个几何对象（向量或线性变换）。 平面上的那个物理点就在那里，一动不动——改变的只是我们描述它的"数字"。

12.1 基与坐标系的多样性

我们最熟悉的直角坐标系，基向量是 î 和 ĵ。但这不是唯一的坐标系。

想象"小明"选了两个倾斜的向量 b₁, b₂ 作为他的基向量——在他的世界里，这两条线构成了他的"倾斜网格"，他同样认为自己用的是标准系。但原点重合。

核心几何事实：无论你用谁的网格，平面上的那一个物理点（向量）就在那里，一动不动。改变的只是我们描述它的"数字"。

12.2 向量基变换：同一向量，不同语言

在你的标准坐标系里，向量表示为 (x, y)。在小明的倾斜坐标系里，它表示为 (x', y')。

问题：怎么把小明说的 (x', y') 翻译成你听得懂的 (x, y)？

需要问小明："你的基向量 b₁, b₂ 在我的直角坐标系里长什么样？"

  • b₁ 在你看来是 (2, 1)
  • b₂ 在你看来是 (-1, 1)

那么，小明眼中的 (x', y')（即 x'·b₁ + y'·b₂），在你眼里就是：

x'·(2,1) + y'·(-1,1) = P · (x', y')

其中 过渡矩阵 P = [b₁, b₂]，每一列就是新基在旧基下的坐标。

几何意义：矩阵 P 就像一个"向导"，把小明网格上的坐标，拉伸旋转，映射回你的标准直角网格上。反向翻译只需要求逆矩阵 P⁻¹。

12.3 核心变换关系

场景操作数学表达
小明坐标 → 你的坐标 左乘过渡矩阵 P v你 = P · v小明
你的坐标 → 小明坐标 左乘逆矩阵 P⁻¹ v小明 = P⁻¹ · v你
关键洞察：向量的分量没变，变的是导航所依赖的"地图"（基）。这与非方阵部分提到的"导航视角"完全一致——变换矩阵的每一列就是新基在旧基下的坐标。

12.4 线性变换的基变换：相似矩阵

同一线性变换，在不同基下对应不同的矩阵。 之前讨论的是"同一个向量在不同基下有不同的坐标"——现在更进一步：对于同一个几何动作（比如旋转90°），在不同坐标系中描述它的矩阵是不同的。

场景：在你的标准坐标系中，变换 A = [[0, -1], [1, 0]] 代表逆时针旋转 90°。小明想在自己倾斜的坐标系中也实现同样的旋转动作——他应该用什么矩阵？

小明手上的向量 v小明 在他的坐标系中描述。要想在你的世界里完成变换，他需要：

  1. 翻译进来：v你 = P · v小明（把小明坐标转为你的坐标）
  2. 执行变换：A · v你（用你的变换矩阵做旋转）
  3. 翻译回去：v小明' = P⁻¹ · (A · v你)（把结果转回小明的语言）
v小明' = (P⁻¹ · A · P) · v小明

所以小明用来描述"旋转90°"这个几何动作的矩阵是：M小明 = P⁻¹ · A · P。

核心公式：若 P 是过渡矩阵（新基在旧基下的坐标），则变换 A 在新基下的矩阵为 P⁻¹ · A · P。这就是相似变换 (Similarity Transformation)。

12.5 具体例子：用数字走一遍

沿用之前的设定：

  • 过渡矩阵 P = [[2, -1], [1, 1]]（小明的基 b₁, b₂ 在你的坐标下的坐标）
  • 你的变换 A = [[0, -1], [1, 0]]（逆时针旋转 90°）
  • 小明有一个向量 v小明 = (1, 0)（在他的世界里就是沿着 b₁ 方向走 1 步）
步骤操作矩阵 / 向量结果几何含义
① 翻译 P · v小明 [[2,-1],[1,1]]·(1,0) (2, 1) 小明说的 b₁ 方向，在你眼里是 (2, 1)——P 的第一列
② 旋转 A · v你 [[0,-1],[1,0]]·(2,1) (-1, 2) 在你的坐标系中完成逆时针 90° 旋转
③ 翻译回 P⁻¹ · (-1, 2) [[1/3,1/3],[-1/3,2/3]]·(-1,2) (1/3, 5/3) 把旋转后的结果告诉小明——用他的语言描述

验证：直接用小明的矩阵 M小明 = P⁻¹AP 计算：

M小明 = P⁻¹ · A · P = [[1/3, -2/3], [5/3, -1/3]]

M小明 · v小明 = (1/3, 5/3) ✓ 结果与三步法一致。

12.6 三大操作的矩阵一览

操作矩阵几何含义
过渡矩阵 P [[b₁, b₂]] 新基在旧基下的坐标。把"新语言"翻译成"旧语言"。
逆矩阵 P⁻¹ [[b₁, b₂]]⁻¹ 把"旧语言"翻译回"新语言"。
相似矩阵 P⁻¹AP P⁻¹ · A · P 同一个线性变换 A 在新基下的矩阵表示。
关键理解：不是换了基向量后变换本身变了——变换（几何动作）始终是同一个（旋转 90°），只是在不同的坐标系中，描述这个动作的"数字"（矩阵）不同了。这就是"相似矩阵"的几何本质。

12.7 总结与思考

  • 同一个向量，在不同的基下有不同的坐标表示
  • 过渡矩阵 P 的列 = 新基在旧基下的坐标
  • 基变换 = 换语言，对象本身没变
  • 逆矩阵 P⁻¹ 做反向翻译

十三、特征向量和特征值

特征向量就是在矩阵变换后方向保持完全不变（或者恰好完全相反）的那些特殊向量。 特征值则描述了这些向量在变换中被拉伸或压缩的比例。

13.1 直觉：为什么会有"方向不变"的向量？

想象你桌上有一块印有网格的正方形橡皮桌布。现在你把双手放在它的左右两侧，用力往外水平拉伸它，使它的宽度变成原来的 2 倍，但垂直方向保持不变。

  • 水平方向的向量：它们的长度变成了原来的 2 倍，但方向完全没变。所以，所有水平方向的向量都是这个变换的特征向量，对应的特征值是 2。
  • 垂直方向的向量：它们的长度没变（缩放倍数是 1），方向也没变。所以，垂直方向的向量也是特征向量，对应的特征值是 1。
  • 斜对角方向的向量：在拉伸后，它们不仅变长了，角度也向水平方向倾斜了（方向改变了）。因此，斜对角方向的向量不是这个变换的特征向量。

13.2 核心公式

Av = λv
  • A：线性变换（矩阵）
  • v：特征向量（方向不变）
  • λ（lambda）：特征值（缩放倍数）

在线性变换的过程中，绝大部分的向量都会改变长度和方向。然而，在整个空间被扭曲的过程中，总有一些特殊的向量非常固执——矩阵 A 乘以它们，仅仅是把它们拉长或缩短了 λ 倍，方向纹丝不动。

关键理解：特征向量告诉你变换的"主轴"在哪里，特征值告诉你沿着这些主轴的缩放力度。

13.3 并非所有变换都有特征向量

二维线性变换不一定有特征向量。

例子：旋转 90° —— 每个向量的长度虽然没变，但是方向都变了。没有任何一个向量能保持方向不变（或完全相反）。所以旋转 90° 的变换在实数域没有特征向量。

13.4 特征基：当基向量恰好是特征向量

如果一个矩阵的所有基向量恰好都是特征向量，那么它的矩阵形式就是对角矩阵：

D = [[λ₁, 0], [0, λ₂]]
  • 对角线上的元素 λ₁, λ₂ 就是对应的特征值
  • 非对角线元素全是 0，意味着各轴之间没有耦合——每个轴独立缩放
  • 对角矩阵的运算（乘法、幂、求逆）极其简单——只是对每个特征值分别操作

13.5 特征基的力量：对角化

假设某个变换的特征向量足够多，使得它们的线性组合能张成整个空间（比如二维平面），那么就可以用这些特征向量作为新的基——即特征基。

A = P · D · P-1
  • P：特征向量作为列组成的过渡矩阵
  • D：特征值组成的对角矩阵
  • P-1：把结果翻译回标准坐标系

为什么这是强大的？在标准坐标系中，很多操作（比如计算 A100）可能非常复杂。但如果把坐标系切换到特征基，变换变成了对角矩阵 D——幂次运算就只是对每个特征值分别求幂：

A100 = P · D100 · P-1

这与基变换章节中"标准系做操作 → 转去新坐标系运算 → 转回来"的思想完全一致。

总结：特征向量找到了变换中"方向不变"的骨架，特征基把复杂运算化简为对角矩阵上的独立运算。这就是相似对角化的几何本质——"转到对的坐标系，问题自然变简单"。

十四、抽象向量空间

抽象向量空间将"向量"的概念从几何箭头推广到了更广泛的数学对象上。 只要一个集合里的元素满足加法和数乘的规则，它们就可以被称为"向量"——无论它们长得像箭头、函数、矩阵、还是多项式。

14.1 到底什么是向量？

  • 几何视角：平面（或高维空间）中的一组箭头？但 4 维以上难以想象。
  • 数值视角：实数组（坐标对）？高维可以用数字理解。
  • 抽象视角：在线性代数中，抽象向量空间是将熟悉的二维平面、三维空间中的"几何向量"概念，推广到更广泛、更普遍的对象上的一种数学结构。

14.2 向量空间的公理

只要一个集合里的元素满足两套规则，它就是一个向量空间，里面的元素就都可以被称为"向量"：

① 加法封闭性

任意两个元素相加，结果仍在集合内。加法满足交换律、结合律，存在零元素和相反元素。

② 数乘封闭性

任意元素乘以一个实数（标量），结果仍在集合内。数乘满足分配律和结合律。

关键洞察：这些规则只涉及"加法"和"缩放"——与元素的具体形状无关。只要满足这两条，函数、矩阵、多项式、信号……都可以被视为"向量"。

14.3 为什么需要抽象？

数学家发现 很多不同类型的东西——比如函数、矩阵、多项式、甚至信号——它们的"加法"和"缩放"操作，和普通的几何向量惊人地相似。例如：

  • 函数可以相加：f + g 定义为一个新函数 (f+g)(x) = f(x) + g(x)
  • 函数可以缩放：a·f 定义为一个新函数 (a·f)(x) = a·f(x)
  • 矩阵和多项式同样有类似的加法和数乘规则

为了不用对每种东西都单独写一套理论，数学家把这些对象的共同特征提取出来，建立了一个统一的"游戏规则"。

意义：抽象向量空间让你可以用线性代数的工具去处理任何满足"加法 + 缩放"规则的数学对象——从解微分方程到信号处理，从量子力学到机器学习，这套框架无处不在。
